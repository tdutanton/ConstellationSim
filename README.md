# ConstellationSim

## Запуск

ПО содержит три микросервиса:

- space-operation-center - основной сервис, хранит все спутники;
- mission-scheduler - сервис запуска миссий по расписанию;
- telemetry-service - сервис по отправке телеметрии на space-operation-center при помощи kafka и
  gRPC.

Все сервисы запускаются в Docker-контейнерах.

**Запуск только основного сервиса и зависимостей для возможности проверки автотестов из другого
проекта (autotest-constellation-sim):**

```bash 
docker compose upd -d server
```

Для запуска всех сервисов:

```bash 
docker compose upd -d
```

При условии запущенного Docker.

# Спутниковая группировка — демо-симулятор

Java-проект, демонстрирующий работу с абстрактными спутниками и их группировками.

## Описание

Проект моделирует:

- абстрактный спутник (`Satellite`);
- два типа спутников:
    - **Связной** (`CommunicationSatellite`) — передаёт данные;
    - **Дистанционного зондирования Земли** (`ImagingSatellite`) — делает снимки;
- группировку спутников (`SatelliteConstellation`), которая может активировать спутники и запускать
  выполнение миссий.

Основная логика запускается из класса `Main`.

## Возможности

- Создание и активация спутников (при достаточном заряде батареи);
- Выполнение миссий с учётом расхода энергии;
- Автоматическая деактивация при разряде батареи;
- Вывод подробной информации в консоль (опционально).
