# ConstellationSim

# Нагрузочное тестирование ConstellationSim

## Описание сценария

Тест имитирует работу пользователя, выполняющего полный бизнес-сценарий:

1. **GET /api/overview** - просмотр текущего состояния группировок (чтение)
2. **POST /api/add-satellites** - создание новой группировки с двумя спутниками (COMMUNICATION +
   IMAGE) (запись)
3. **POST /api/activate-satellites/{name}** - активация спутников в группировке (обновление)
4. **POST /api/missions** - выполнение миссии на всю группировку (запись)
5. **GET /api/overview** - повторный просмотр для верификации изменений (чтение)
6. **DELETE /api/delete-satellite/...** - удаление одного спутника (удаление)

Каждый виртуальный пользователь использует уникальные имена, чтобы избежать конфликтов.

## Требования

- [k6](https://grafana.com/docs/k6/latest/set-up/) (установка: `winget install k6` или
  `brew install k6`)
- Запущенный сервер ConstellationSim (`docker compose up -d`)

## Запуск

Перейти в папку load-tests и выполнить:

```bash
k6 run ./scenario-test.js -e BASE_URL=http://localhost:8080
```

## Параметры теста

| Параметр         | Значение                     |
|------------------|------------------------------|
| Инструмент       | k6                           |
| Профиль нагрузки | 10 → 20 → 50 → 100 VU (ramp) |
| Длительность     | ~80 секунд                   |
| Действия         | GET + POST + DELETE          |
| Отчет            | HTML (built-in k6)           |

## Отчет

После завершения теста будет сгенерирован HTML-отчет `load-test-report.html`. Находится в папке
load-tests.

## Описание проекта

ПО содержит три микросервиса:

- space-operation-center - основной сервис, хранит все спутники;
- mission-scheduler - сервис запуска миссий по расписанию;
- telemetry-service - сервис по отправке телеметрии на space-operation-center при помощи kafka и
  gRPC.

Все сервисы запускаются в Docker-контейнерах.

**Запуск только основного сервиса и зависимостей для возможности проверки автотестов из другого
проекта (autotest-constellation-sim):**

```bash 
docker compose upd -d server
```

Для запуска всех сервисов:

```bash 
docker compose upd -d
```

При условии запущенного Docker.

# Спутниковая группировка — демо-симулятор

Java-проект, демонстрирующий работу с абстрактными спутниками и их группировками.

## Описание

Проект моделирует:

- абстрактный спутник (`Satellite`);
- два типа спутников:
    - **Связной** (`CommunicationSatellite`) — передаёт данные;
    - **Дистанционного зондирования Земли** (`ImagingSatellite`) — делает снимки;
- группировку спутников (`SatelliteConstellation`), которая может активировать спутники и запускать
  выполнение миссий.

Основная логика запускается из класса `Main`.

## Возможности

- Создание и активация спутников (при достаточном заряде батареи);
- Выполнение миссий с учётом расхода энергии;
- Автоматическая деактивация при разряде батареи;
- Вывод подробной информации в консоль (опционально).
